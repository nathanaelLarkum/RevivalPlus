1. users Table
Handles user registration and profile information.

id (Primary Key)

name (String)

email (String, Unique)

password (String)

phone_number (String, Nullable)

last_known_latitude (Decimal, Nullable)

last_known_longitude (Decimal, Nullable)

created_at, updated_at (Timestamps)

2. countries Table (New)
A lookup table for all countries to ensure data integrity.

id (Primary Key)

name (String, Unique) - e.g., "United States", "Germany".

code (String, Unique) - The 2-letter ISO code, e.g., "US", "DE".

3. denominations Table
A lookup table for categorizing churches.

id (Primary Key)

name (String, Unique)

slug (String, Unique)

description (Text, Nullable)

created_at, updated_at (Timestamps)

4. tags Table (Replaces amenities)
A flexible, categorized lookup table for all church attributes and filter options.

id (Primary Key)

name (String) - e.g., "Modern", "English", "Youth Group".

slug (String, Unique)

category (String) - e.g., "service-style", "language", "program".

created_at, updated_at (Timestamps)

5. event_types Table
A lookup table for categorizing all church events.

id (Primary Key)

name (String, Unique) - e.g., "Morning Service", "Youth Night".

slug (String, Unique)

created_at, updated_at (Timestamps)

6. churches Table (Revised)
The central table of the application with updated, international-friendly address fields.

id (Primary Key)

name (String)

denomination_id (Foreign Key) - Links to the denominations table.

country_id (Foreign Key) - Links to the countries table.

address_line_1 (String)

address_line_2 (String, Nullable)

city (String)

state_province_region (String)

postal_code (String)

latitude (Decimal)

longitude (Decimal)

email (String, Nullable)

website_url (String, Nullable)

instagram_url (String, Nullable)

facebook_url (String, Nullable)

timezone (String)

created_at, updated_at (Timestamps)

7. reviews Table
Contains user-generated ratings and comments for churches.

id (Primary Key)

user_id (Foreign Key)

church_id (Foreign Key)

rating (Integer)

title (String, Nullable)

body (Text)

status (String, default: 'pending')

8. events Table
The robust and searchable schedule system for every recurring event at each church.

id (Primary Key)

church_id (Foreign Key)

event_type_id (Foreign Key)

day_of_week (Integer)

start_time (Time)

end_time (Time, Nullable)

description (Text, Nullable)

9. church_tag Table (Replaces amenity_church)
The pivot table connecting churches to their various tags.

church_id (Foreign Key)

tag_id (Foreign Key)
